### 🎯 Tujuan Belajar
Mata kuliah Reverse Engineering ini merupakan pilar penting dalam konsentrasi Keamanan Sistem. Target utama saya dalam mempelajari mata kuliah ini adalah mampu memahami cara kerja biner aplikasi tanpa memiliki akses ke *source code* aslinya. Secara spesifik, saya ingin menguasai pembacaan bahasa Assembly x86/x64, mahir menggunakan *tools* analisis seperti Ghidra dan x64dbg, serta mampu mengidentifikasi algoritma ataupun malware melalui analisis statis dan dinamis. Selain aspek teknis, saya juga bertujuan untuk memahami batasan etika dan hukum (seperti UU ITE) agar keahlian RE ini dapat saya gunakan secara bertanggung jawab dalam dunia profesional.

## 🗓️ Catatan Jurnal Mingguan

### 🟢 Minggu 1: Pengenalan & Rencana Pembelajaran Semester (RPS)
**Tanggal:** [Isi Tanggal Kuliah Minggu Pertama Kamu, misal: 1 September 2026]

Pada minggu pertama ini, saya mempelajari orientasi perkuliahan dan rencana pembelajaran semester (RPS) untuk mata kuliah Reverse Engineering, termasuk pengenalan dasar mengenai ruang lingkup, sejarah, serta aspek etika dan legalitas dalam membongkar file biner. Saya memahami bahwa mata kuliah ini berfokus pada analisis perangkat lunak tanpa kode sumber melalui metode statis dan dinamis, serta bagaimana hukum seperti UU ITE di Indonesia membatasi atau melegalkan aktivitas ini untuk tujuan interoperabilitas dan keamanan. Namun, saya masih merasa bingung dan sedikit gentar melihat tumpukan materi teknis ke depan, terutama mengenai representasi memori *low-level* seperti register, stack, dan *calling convention* bahasa Assembly yang tampaknya membutuhkan ketelitian sangat tinggi.
